# 基于Hexo与GitHub Actions搭建个人博客实践总结
## 一、实践整体流程
本次课程实践选用Hexo开源静态博客框架，搭配GitHub Actions实现自动化部署，完整流程分为四部分：
1. 本地搭建Hexo环境，使用Markdown撰写博客文章；
2. 创建公开GitHub仓库，存放博客全部源码；
3. 配置GitHub Actions流水线，代码推送后自动构建静态网页；
4. 依托GitHub Pages对外发布可公网访问的博客网站。

## 二、工具作用理解
1. Hexo：将Markdown文本编译成纯静态HTML页面，轻量化、无需后端服务器；
2. GitHub仓库：代码版本管理，保存博客源码、文章、主题配置；
3. GitHub Actions：CI/CD自动化工具，替代手动打包、上传页面，推送代码自动更新网站；
4. GitHub Pages：免费静态网页托管服务，生成本人专属公网访问域名。
